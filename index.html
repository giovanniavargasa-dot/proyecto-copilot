Post #1: Introducción a los Grafos: Nodos, Aristas y Tipos

Un grafo es una estructura matemática que representa relaciones entre objetos. Está formado por un conjunto de vértices (o nodos) y un conjunto de aristas (o ejes) que conectan pares de vértices.

Definición Formal

Un grafo G se define como un par ordenado G = (V, E), donde:

  • V es un conjunto de vértices (nodos).
  • E es un conjunto de aristas (pares de vértices).

Conceptos Clave

  • Vértices (Nodos): Elementos o puntos que representan entidades (personas, ciudades, computadoras, etc.).
  • Aristas (Ejes): Conexiones entre pares de vértices que representan relaciones (amistad, distancia, conexión de red, etc.).
  • Grado: Número de aristas conectadas a un vértice.

Tipos de Grafos

  • Dirigidos (Dígrafos): Las aristas tienen dirección. Una arista (A → B) es diferente de (B → A). Se utilizan para representar flujos y relaciones asimétricas.
  • No Dirigidos: Las aristas no tienen dirección. Una conexión entre A y B es mutua. Se utilizan para representar relaciones simétricas.
  • Ponderados: Las aristas tienen un valor (peso) asociado, como distancia, costo o capacidad.
  • Cíclicos: Contienen al menos un ciclo (camino que vuelve al vértice de inicio).
  • Acíclicos: No contienen ciclos. Ejemplo: árbol, DAG (Directed Acyclic Graph).

Post #2: Representación de Grafos en Memoria

Existen dos formas principales de representar un grafo en la memoria de una computadora, cada una con ventajas y desventajas según el contexto de uso.

1. Lista de Adyacencia

Concepto

Una lista de adyacencia es una colección de listas vinculadas (o arreglos) donde cada vértice tiene una lista de sus adyacentes (vértices con los que está conectado).

Ventajas

  • Eficiencia de espacio: Ideal para grafos dispersos (pocos aristas). Ocupa O(V + E) espacio.
  • Recorrido eficiente: Rápido acceso a los vecinos de un vértice.
  • Flexibilidad: Fácil de implementar y modificar.

Desventajas

  • Consulta lenta: Verificar si existe una arista entre dos vértices específicos es O(V) en el peor caso.

Ejemplo: Lista de Adyacencia

Grafo:
Vértice 1: [2, 4, 5]
Vértice 2: [1, 3, 4]
Vértice 3: [2, 4]
Vértice 4: [1, 2, 3, 5]
Vértice 5: [1, 4]

2. Matriz de Adyacencia

Concepto

Una matriz de adyacencia es una matriz cuadrada de tamaño V × V, donde el elemento (i, j) indica si existe una arista entre los vértices i y j.

Ventajas

  • Consulta rápida: Verificar si existe una arista es O(1) en tiempo constante.
  • Operaciones matriciales: Permite usar operaciones algebraicas para análisis de grafos.

Desventajas

  • Ineficiencia de espacio: Ocupa O(V²) espacio, incluso para grafos dispersos.
  • Lentitud en recorridos: Recorrer todos los vecinos de un vértice es O(V).

Ejemplo: Matriz de Adyacencia

1 2 3 4 5
1 0 1 0 1 1
2 1 0 1 1 0
3 0 1 0 1 0
4 1 1 1 0 1
5 1 0 0 1 0

Resumen de Eficiencia

Operación Lista de Adyacencia Matriz de Adyacencia
Espacio O(V + E) O(V²)
Agregar arista O(1) O(1)
Eliminar arista O(V) O(1)
Buscar arista (i,j) O(V) O(1)
Recorrer vecinos O(grado) O(V)

¿Cuál Usar?

  • Lista de Adyacencia: Para grafos dispersos (E << V²) o cuando necesitas recorrer eficientemente los vecinos.
  • Matriz de Adyacencia: Para grafos densos (E ≈ V²) o cuando necesitas búsquedas rápidas de aristas.

Post #3: Algoritmos Fundamentales de Recorrido en Grafos

Los algoritmos de recorrido (o búsqueda) son fundamentales en la teoría de grafos. Los dos más importantes son la Búsqueda en Amplitud (BFS) y la Búsqueda en Profundidad (DFS).

1. Búsqueda en Amplitud (BFS - Breadth-First Search)

Concepto

BFS es un algoritmo que explora un grafo "por niveles" o "por capas", comenzando desde un vértice inicial. Visita todos los vértices a distancia k antes de visitar los de distancia k+1.

Características

  • Estructura de datos: Utiliza una cola (Queue) para almacenar los vértices a explorar.
  • Complejidad temporal: O(V + E) donde V es el número de vértices y E el número de aristas.
  • Complejidad espacial: O(V) para la cola y la matriz de visitados.
  • Orden de exploración: Primero los adyacentes más cercanos.

Pseudocódigo

BFS(grafo, inicio):
  cola = nueva Cola()
  visitados = conjunto vacío
  cola.encolar(inicio)
  visitados.agregar(inicio)

  mientras cola no esté vacía:
    vértice = cola.desencolar()
    procesar(vértice)
    para cada adyacente en vértice:
      si adyacente no está visitado:
        visitados.agregar(adyacente)
        cola.encolar(adyacente)

Ejemplo de Ejecución

Orden de visita en BFS comenzando desde el nodo 1:

Paso 1: Cola = [1], Visitados = {1}
Paso 2: Procesa 1, agrega [2, 4, 5]. Cola = [2, 4, 5], Visitados = {1, 2, 4, 5}
Paso 3: Procesa 2, agrega [3]. Cola = [4, 5, 3], Visitados = {1, 2, 4, 5, 3}
Paso 4: Procesa 4 (sin nuevos adyacentes). Cola = [5, 3]
Paso 5: Procesa 5 (sin nuevos adyacentes). Cola = [3]
Paso 6: Procesa 3 (sin nuevos adyacentes). Cola = []
Orden final de visita: 1 → 2 → 4 → 5 → 3

Aplicaciones de BFS

  • Encontrar el camino más corto en un grafo no ponderado.
  • Búsqueda por niveles.
  • Verificar si un grafo es conexo.
  • Análisis de redes sociales.

2. Búsqueda en Profundidad (DFS - Depth-First Search)

Concepto

DFS es un algoritmo que explora un grafo "en profundidad", avanzando lo más lejos posible en cada rama antes de retroceder. Utiliza un enfoque recursivo o iterativo con una pila.

Características

  • Estructura de datos: Utiliza una pila (Stack) o recursión para almacenar los vértices a explorar.
  • Complejidad temporal: O(V + E).
  • Complejidad espacial: O(V) en la pila de recursión o estructura de datos.
  • Orden de exploración: Profundiza lo máximo posible antes de retroceder.

Pseudocódigo (Versión Recursiva)

DFS(grafo, vértice, visitados):
  visitados.agregar(vértice)
  procesar(vértice)

  para cada adyacente en vértice:
    si adyacente no está visitado:
      DFS(grafo, adyacente, visitados)

Llamada inicial:
  visitados = conjunto vacío
  DFS(grafo, inicio, visitados)

Pseudocódigo (Versión Iterativa)

DFS_Iterativo(grafo, inicio):
  pila = nueva Pila()
  visitados = conjunto vacío
  pila.apilar(inicio)
  visitados.agregar(inicio)

  mientras pila no esté vacía:
    vértice = pila.desapilar()
    procesar(vértice)
    para cada adyacente en vértice:
      si adyacente no está visitado:
        visitados.agregar(adyacente)
        pila.apilar(adyacente)

Ejemplo de Ejecución

Orden de visita en DFS comenzando desde el nodo 1 (versión recursiva):

Paso 1: Visita 1, explora adyacentes [2, 4, 5]
Paso 2: Visita 2 (primer adyacente), explora adyacentes [1, 3, 4] → 1 ya visitado, intenta 3
Paso 3: Visita 3, explora adyacentes [2, 4] → 2 ya visitado, intenta 4
Paso 4: Visita 4, explora adyacentes [1, 2, 3, 5] → 1, 2, 3 ya visitados, intenta 5
Paso 5: Visita 5, explora adyacentes [1, 4] → ambos ya visitados, retrocede
Orden final de visita: 1 → 2 → 3 → 4 → 5

Aplicaciones de DFS

  • Detectar ciclos en un grafo.
  • Ordenamiento topológico (en DAGs).
  • Detectar componentes fuertemente conectadas.
  • Resolver problemas de laberinto y backtracking.
  • Análisis de conectividad.

Comparación: BFS vs DFS

Aspecto BFS DFS
Estructura de datos Cola (Queue) Pila (Stack) / Recursión
Complejidad temporal O(V + E) O(V + E)
Complejidad espacial O(V) O(V)
Exploración Por niveles (ancho) En profundidad (largo)
Camino más corto (no ponderado) ✓ Garantizado ✗ No garantizado
Detectar ciclos Posible ✓ Más eficiente
Uso típico Distancias mínimas, análisis de niveles Ciclos, conectividad, ordenamiento topológico

Resumen

BFS es ideal cuando necesitas explorar el grafo por niveles y encontrar caminos más cortos. DFS es superior para análisis de conectividad, detección de ciclos y problemas de backtracking. Ambos tienen la misma complejidad temporal O(V + E) pero difieren en el orden de exploración y aplicaciones.
